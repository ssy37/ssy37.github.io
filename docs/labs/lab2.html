Objective

Lab 2 introduces the IMU. Through the tasks, data is collected via the IMU's accelerometer and gyroscope. Various filtering techniques are used to get clean, accurate signals. In this lab, we also received the physical robot and performed stunts with it.

Setup

AD0_VAL

The code comment reads, “On the SparkFun 9DoF IMU breakout the default is 1, and when the ADR jumper is closed the value becomes 0.” According to lecture 4 code, it is the value of the last bit on the I2C address. It is 1 right now because the AD0 pin doesn't have anything soldered to it.

Accelerometer

After validating that the IMU was set up correctly, I collected accelerometer data to compute pitch and roll angles (recall that the accelerometer cannot detect yaw). Note that for this task, the roll/pitch conventions follow the convention in the lecture 4 demo code (pitch about x, roll about y). I change this convention in the next task. To collect the data, I rotated the sensor mostly about its y-axis. Following the convention I used for this task, I expected pitch to be ~0.

Fourier and Low-Pass Filter

The raw accelerometer data is very noisy; applying a low-pass filter to the data can clean up the signal. Before implementing the LPF, I needed to determine the best cutoff frequency for the filter. Using a high cutoff frequency means more high-frequency signals are kept. Noise tends to be high frequency, so having a high cutoff frequency means that you might be keeping noise. Having a very low cutoff frequency will keep only low-frequency signals, resulting in super smooth but potentially incomplete signals.

To determine the cutoff, I analyzed the signal’s frequency spectrum by performing a Fourier transform on my data.

The raw data plots show approximately 2 periods in 3 seconds, which means that the main signal’s frequency is ~0.66Hz. The frequency spectrum corroborates this, as there is spike at approximately 0.66Hz. We also see a smaller spike at approximately 1.25Hz-1.5Hz. From this, I know that the frequencies I keep need to include 0.66Hz, and should include 1.25Hz-1.5Hz as well.
